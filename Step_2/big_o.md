# 시간복잡도란?
시간 복잡도는 알고리즘의 성능을 나타내는 지표로 입력 크기에 대한 연산 횟수의 상한을 의미한다.
시간 복잡도는 낮으면 낮을수록 좋다. 

## 알고리즘 수행 시간을 측정하는 방법
- 절대 시간을 측정하는 방법
    - 말 그대로 시간을 측정하는 방법
    - 예를 들어 배열에서 검색하는 프로그램을 작성한 다음 프로그램을 실행하여 결과가 나올 때까지의 시간을 측정하면 된다.
    - 이 방법은 프로그램을 실행하는 환경에 따라 달라질 수 있어 코딩 테스트에서는 잘 활용하지 않는다
- 시간 복잡도를 측정하는 방법
    - 알고리즘이 시작한 순간부터 결괏값이 나올 때까지의 연산 횟수를 나타낸다.
    - 시간 복잡도를 측정한 결과는 다음과 같이 최선(best), 평균(average), 최악(worst)의 경우로 나뉜다

## 시간 복잡도를 표현하는 방법
- 시간 복잡도를 최선, 최악 이렇게 특정한 입력 크기에 따른 연산 횟수로 시간 복잡도를 이야기하는 것은 특정 상황에 한한 것이므로 무의미하다.
- 예를 들어 1차원 배열 검색하는 문제에서 배열 크기가 1이면 최선, 보통, 최악의 경우는 모두 연산 횟수는 1이다. 이 결과만 보면 연산 횟수가 1인 성능을 가지는 알고리즘이구나 생각하면 안된다.
- 즉 특정 입력 크기에 한하여 연산 횟수 기준으로 복잡도를 측정하면 안된다. 입력 크기를 N으로 일반화하여 연산 횟수의 추이를 나타내야 한다.
- 이러한 방식으로 입력 크기에 따른 연산 횟수의 추이를 활용해서 시간 복잡도를 표현하는 방법을 점근적 표기법이라고 한다.
- 코딩 테스트에서는 모든 경우의 수에서 알고리즘이 문제를 처리하는 것을 고려해야 하므로 시간 복잡도는 최악의 경우를 가정하여 이야기하는 것이 일반적이다.

# Big-O 표기법
최악의 경우에 시간 복잡도를 표현하는 방법은 점근적 표기법인 상한선을 활용하는 방법이다. 그리고 이 방법은 big-O 표기법이라고 한다. big-O 표기법은 어떤 프로그램의 연산 횟수가 f(x)라고 할 때 함수의 최고차항만 남기고 계수를 지워 O(...)와 같이 표기한다.

## 최고차항만 남기고 차수를 지우는 이유
f(x)의 최고차항만 남기고 계수를 지워 O(...)와 같이 쓸수 있다고 하였다.

입력 크기 n이 충분히 커지면 최고차항 이외의 항은 영향이 미미해지기 때문이다.

예를 들어 연산 횟수가 `3n² + 5n + 10`인 알고리즘이 있다고 하자.

| n | 3n² | 5n | 10 | 전체 |
|---|-----|----|----|------|
| 10 | 300 | 50 | 10 | 360 |
| 100 | 30,000 | 500 | 10 | 30,510 |
| 10,000 | 300,000,000 | 50,000 | 10 | 300,050,010 |

n이 커질수록 `5n`과 `10`은 `3n²`에 비해 무시할 수 있을 정도로 작아진다. 또한 계수 `3`도 증가 추이에는 영향을 주지 않으므로 제거한다. 따라서 `3n² + 5n + 10` → **O(n²)** 로 표기한다.

### Big-O 표기 규칙

1. **상수항 제거** : `O(3n)` → `O(n)`
2. **최고차항만 남기기** : `O(n² + n)` → `O(n²)`
3. **계수 제거** : `O(5n²)` → `O(n²)`

### 변환 예시

| 연산 횟수 | Big-O |
|----------|-------|
| `7` | O(1) |
| `3n + 5` | O(n) |
| `2n² + 3n + 1` | O(n²) |
| `5·2ⁿ + n³` | O(2ⁿ) |

## 주요 시간복잡도 정리

빠른 순서대로 나열하면 다음과 같다.

**O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(2ⁿ) < O(n!)**

### O(1) — 상수 시간

입력 크기와 관계없이 항상 일정한 연산 횟수를 가진다.

```python
# 배열의 인덱스 접근
arr = [1, 2, 3, 4, 5]
print(arr[0])  # 항상 한 번의 연산
```

### O(log n) — 로그 시간

연산마다 탐색 범위가 절반으로 줄어드는 경우이다. 대표적으로 이진 탐색이 있다.

```python
# 이진 탐색
def binary_search(arr, target):
    left, right = 0, len(arr) - 1
    while left <= right:
        mid = (left + right) // 2
        if arr[mid] == target:
            return mid
        elif arr[mid] < target:
            left = mid + 1
        else:
            right = mid - 1
    return -1
```

### O(n) — 선형 시간

입력 크기에 비례하여 연산 횟수가 증가한다. 반복문 1개가 대표적이다.

```python
# 배열에서 최대값 찾기
def find_max(arr):
    max_val = arr[0]
    for val in arr:       # n번 반복
        if val > max_val:
            max_val = val
    return max_val
```

### O(n log n) — 선형 로그 시간

효율적인 정렬 알고리즘의 시간복잡도이다. 파이썬의 `sort()`가 이에 해당한다.

```python
arr = [5, 3, 1, 4, 2]
arr.sort()  # Timsort — O(n log n)
```

### O(n²) — 이차 시간

이중 반복문이 대표적이다. 입력이 커지면 급격히 느려진다.

```python
# 버블 정렬
def bubble_sort(arr):
    n = len(arr)
    for i in range(n):            # n번
        for j in range(n - 1):    # n번 → 총 n × n = n²
            if arr[j] > arr[j + 1]:
                arr[j], arr[j + 1] = arr[j + 1], arr[j]
```

### O(2ⁿ) — 지수 시간

재귀 호출이 2갈래로 나뉘는 경우이다. 입력이 조금만 커져도 연산 횟수가 폭발적으로 증가한다.

```python
# 피보나치 (비효율적 재귀)
def fib(n):
    if n <= 1:
        return n
    return fib(n - 1) + fib(n - 2)  # 호출이 2배씩 늘어남
```

# 복잡도 비교/판별 연습

코드를 보고 시간복잡도를 판별하는 연습을 해보자.

## 예제 1 — 단일 반복문

```python
def example1(n):
    total = 0
    for i in range(n):
        total += i
    return total
```

→ 반복문 1개, n번 반복 → **O(n)**

## 예제 2 — 이중 반복문

```python
def example2(n):
    count = 0
    for i in range(n):
        for j in range(n):
            count += 1
    return count
```

→ 바깥 n번 × 안쪽 n번 → **O(n²)**

## 예제 3 — 반씩 줄어드는 반복문

```python
def example3(n):
    count = 0
    while n > 1:
        n //= 2
        count += 1
    return count
```

→ n이 매번 절반으로 줄어듦 → **O(log n)**

## 예제 4 — 반복문 두 개 (중첩 아님)

```python
def example4(n):
    for i in range(n):
        print(i)
    for j in range(n):
        print(j)
```

→ O(n) + O(n) = O(2n) → **O(n)** (상수 제거)

## 예제 5 — 안쪽 반복 횟수가 줄어드는 경우

```python
def example5(n):
    count = 0
    for i in range(n):
        for j in range(i, n):
            count += 1
    return count
```

→ n + (n-1) + (n-2) + ... + 1 = n(n+1)/2 → **O(n²)**

## Step_1 자료구조 연산 복잡도 복습

| 자료구조 | 접근 | 탐색 | 삽입 | 삭제 |
|---------|------|------|------|------|
| 배열 (list) | O(1) | O(n) | O(n) | O(n) |
| 연결 리스트 | O(n) | O(n) | O(1)* | O(1)* |
| 스택 | O(n) | O(n) | O(1) | O(1) |
| 큐 (deque) | O(n) | O(n) | O(1) | O(1) |

*연결 리스트의 삽입/삭제는 해당 노드를 이미 알고 있을 때 O(1). 탐색이 필요하면 O(n).

# 공간복잡도

공간복잡도는 알고리즘이 실행되는 동안 사용하는 **메모리 양**을 입력 크기 n에 대해 표현한 것이다. 시간복잡도와 마찬가지로 Big-O로 표기한다.

## 공간복잡도 구성 요소

- **고정 공간** : 입력 크기와 무관하게 항상 필요한 공간 (변수, 상수 등)
- **가변 공간** : 입력 크기에 따라 달라지는 공간 (리스트 복사, 재귀 호출 스택 등)

## 예시별 공간복잡도

### O(1) — 상수 공간

변수 몇 개만 사용하며 입력 크기와 무관하다.

```python
def sum_arr(arr):
    total = 0          # 변수 1개
    for val in arr:
        total += val
    return total
# 입력 배열 외에 추가 메모리 = 변수 1개 → O(1)
```

### O(n) — 선형 공간

입력 크기만큼 추가 공간을 사용한다.

```python
def copy_arr(arr):
    new_arr = []
    for val in arr:
        new_arr.append(val)  # n개의 요소를 복사
    return new_arr
# 추가 메모리 = n개 → O(n)
```

### O(n) — 재귀 호출 스택

재귀 호출도 호출 스택에 메모리를 사용한다.

```python
def factorial(n):
    if n <= 1:
        return 1
    return n * factorial(n - 1)
# 재귀 깊이 = n → 호출 스택 O(n)
```

### O(n²) — 2차원 배열 생성

```python
def make_matrix(n):
    matrix = [[0] * n for _ in range(n)]
    return matrix
# n × n 크기 → O(n²)
```

## 시간 vs 공간 트레이드오프

시간복잡도와 공간복잡도는 서로 반비례 관계인 경우가 많다. 메모리를 더 사용하면 시간을 줄일 수 있고, 메모리를 아끼면 시간이 더 걸릴 수 있다.

```python
# 방법 1: 시간 O(n²), 공간 O(1) — 이중 반복문으로 중복 검사
def has_duplicate_v1(arr):
    for i in range(len(arr)):
        for j in range(i + 1, len(arr)):
            if arr[i] == arr[j]:
                return True
    return False

# 방법 2: 시간 O(n), 공간 O(n) — set을 활용하여 중복 검사
def has_duplicate_v2(arr):
    seen = set()
    for val in arr:
        if val in seen:
            return True
        seen.add(val)
    return False
```

방법 2는 `set`에 메모리를 추가로 사용하지만 시간이 O(n²)에서 O(n)으로 크게 줄어든다. 코딩 테스트에서는 보통 **시간을 줄이는 쪽**을 선호한다.

# 코딩 테스트에서의 활용

## 입력 크기로 허용 복잡도 추정하기

코딩 테스트의 시간 제한은 보통 1~2초이다. 파이썬 기준 **1초에 약 2,000만 번**의 단순 연산이 가능하다고 가정하면, 입력 크기 n에 따라 허용되는 시간복잡도를 추정할 수 있다.

| 입력 크기 (n) | 허용 시간복잡도 |
|--------------|---------------|
| n ≤ 10 | O(n!) — 완전 탐색 가능 |
| n ≤ 20 | O(2ⁿ) |
| n ≤ 500 | O(n³) |
| n ≤ 5,000 | O(n²) |
| n ≤ 1,000,000 | O(n log n) |
| n ≤ 10,000,000 | O(n) |
| 그 이상 | O(log n) 또는 O(1) |

## 활용법

1. **문제의 입력 크기(n)를 확인**한다
2. 위 표를 참고하여 **허용되는 시간복잡도를 추정**한다
3. 해당 복잡도에 맞는 **알고리즘/자료구조를 선택**한다

예를 들어 n이 100만이라면 O(n²)은 1조 번 연산으로 시간 초과가 발생한다. O(n log n) 이하의 알고리즘을 선택해야 한다.
